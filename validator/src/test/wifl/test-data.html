WIFL Test Data

This document contains test WIFL data. It describes a resource at http://api.example.com/bogus. There are also some examples.

Every resource in this API has a hexadecimal apikey query parameter. Every resource has a GET request (with a pointless parameter) which may respond with 200 OK, and a application/json body which validates against the JSON Schema.

The bogus API has a child /dogs resource. This resource supports a POST request, uploading JSON, and getting back a 201 Created, with a JSON representation, and a required URI Location header.

The dogs resource has a child /{dogID} resource. The dogID path parameter is a natural number. This resource supports a PUT request, uploading JSON, and getting back a 200 OK, and a DELETE request, getting back a 204 No Content.

The document contains a deliberately broken hyperlink, and a recursive hyperlink.
