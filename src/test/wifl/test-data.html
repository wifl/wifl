WIFL Test Data

This document contains test WIFL data. It describes a resource at http://api.example.com/bogus.

Every resource in this API has a hexadecimal apikey query parameter. Every resource has a GET request, which may respond with 200 OK, and a application/json body which validates against the JSON Schema.

The bogus API has a child /dogs resource.

The dogs resource has a child /{dogID} resource. The dogID path parameter is a natural number.
